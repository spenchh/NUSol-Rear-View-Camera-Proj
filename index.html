NUSolar Rear Vision System

Real-time rear-vision camera + display for ASC 2026 · Section 9.7.E compliance · Spencer Kogoma

Project Summary

Design and build a real-time rear-vision camera and display system for the NUSolar car that passes ASC 2026 Section 9.7.E. Must provide always-on, mirror-image, low-latency rear view. Driver tested on identifying arrow direction at 15m.

Selected System Architecture

Chosen path: Raspberry Pi Camera Module 3 Wide → CSI extension → Raspberry Pi 4B → Waveshare 7" HDMI display.

Raspberry Pi Camera Module 3 Wide
Selected as the final camera because it gives the team a wide field of view, Pi-native support, good outdoor image quality, and a clean software path for mirror-flipped rear vision.
Raspberry Pi 4B + HDMI display
The Pi handles the live mirrored feed and outputs to the driver-facing Waveshare display.
Remaining mechanical decision
The only major decision left is mounting: where the rear camera goes on the car, how the bracket fits, and how the wiring is routed cleanly and safely.

Key Findings

★ Final Choice: Pi Camera Module 3 Wide + CSI Extender
Official Raspberry Pi camera with 12MP IMX708 sensor, 120° wide FOV (~104° horizontal), HDR mode, autofocus, and CSI interface (lowest latency path to Pi GPU). Arducam CSI-to-HDMI extension kit ($15) solves the 3m cable run. Total system cost: ~$108. Selected final camera path for the rear-vision system.
ELP & Arducam USB cameras remain strong alternatives
Both meet all regs, work with Pi 4B, achievable in 3 weeks. ELP OV4689 has proven 60fps@1080p reliability. Arducam B0497 has the best sensor (STARVIS 2) for low-light. Good fallbacks if CSI extender has EMI issues in the motor bay.
AHD kit is the safe fallback
Cheapest, lowest latency, purpose-built for vehicles. But limited quality improvement over what the team has used before.
GoPro is viable if sponsored
Hero 13 HDMI output is good (1080p60, 2-3 frame delay with stabilization off). But needs Media Mod ($80 extra) and too expensive without sponsorship.
Insta360 X5 is not viable for rear-view
Webcam mode is 30fps with stitching latency. HDMI requires manual menu navigation (violates always-on reg). Pursuing for content creation only.

Final Camera Choice

Decision: We chose the Raspberry Pi Camera Module 3 Wide for the rear-vision system. The final direction is a Pi-native camera pipeline using the Module 3 Wide, Raspberry Pi 4B, and Waveshare 7" HDMI display. The remaining work is mechanical integration: deciding exactly where the camera mounts on the car and how the housing/bracket fits.
