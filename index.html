NUSolar Rear Vision System

Project Summary

Design and build a real-time rear-vision camera and display system for the NUSolar car that passes ASC 2026 Section 9.7.E scrutineering. Must provide always-on, mirror-image, low-latency rear view. Driver tested on identifying arrow direction at 15m.

System Architecture — Open Decision

Pi is available but not required. Four viable approaches identified:

Pi-Mediated (USB)
USB camera → Pi 4B → HDMI display. Most flexible, software mirror flip.
★ Pi-Mediated (CSI + HDMI Extender)
CSI camera → Arducam CSI-to-HDMI adapter → 3m HDMI cable → adapter → Pi 4B → HDMI display. Lowest latency path — CSI goes direct to GPU. 3m cable solved by $15 extension kit.
Direct Analog (AHD)
AHD camera → wire → dedicated monitor. Simplest, lowest latency, no Pi.
Action Camera HDMI
GoPro/Insta360 → HDMI direct or via Pi → display. Best quality, highest cost.

FPGA Considered & Ruled Out

Considered using an FPGA for lowest-possible latency (<10ms). Shannon said "design complexity is significantly higher. I'll let you figure out if the benefits for latency + quality are better." After analysis: Mach XO3L lacks resources for video. A capable FPGA board would cost $100-250+ and require weeks of HDL development beyond current skill level. Pi path achieves 50-80ms which is sufficient for rear-view at highway speeds. Decision: Pi for now, FPGA as documented future upgrade path.

Key Findings

★ Pi Camera Module 3 Wide + CSI Extender is the top recommendation
Discovered during independent deep research — official Raspberry Pi camera with 12MP IMX708 sensor, 120° wide FOV (~104° horizontal), HDR mode, autofocus, and CSI interface (lowest latency path to Pi GPU). Arducam CSI-to-HDMI extension kit ($15) solves the 3m cable run. Total system cost: ~$108. Best value, best latency, best software support.
ELP & Arducam USB cameras remain strong alternatives
Both meet all regs, work with Pi 4B, achievable in 3 weeks. ELP OV4689 has proven 60fps@1080p reliability. Arducam B0497 has the best sensor (STARVIS 2) for low-light. Good fallbacks if CSI extender has EMI issues in the motor bay.
AHD kit is the safe fallback
Cheapest, lowest latency, purpose-built for vehicles. But limited quality improvement over what the team has used before.
GoPro is viable if sponsored
Hero 13 HDMI output checks out (1080p60, 2-3 frame delay with stabilization off). But needs Media Mod ($80 extra) and too expensive without sponsorship.
Insta360 X5 is not viable for rear-view
Webcam mode = 30fps with stitching latency. HDMI requires manual menu navigation (violates always-on reg). Pursue for content creation only.

Arducam CSI-to-HDMI extension kit ($15) solves the 3m cable run. Total system cost: ~$108. Best value, best latency, best software support.
ELP & Arducam USB cameras remain strong alternatives
Both meet all regs, work with Pi 4B, achievable in 3 weeks. ELP OV4689 has proven 60fps@1080p reliability. Arducam B0497 has the best sensor (STARVIS 2) for low-light. Good fallbacks if CSI extender has EMI issues in the motor bay.
AHD kit is the safe fallback
Cheapest, lowest latency, purpose-built for vehicles. But limited quality improvement over what the team has used before.
GoPro is viable if sponsored
Hero 13 HDMI output checks out (1080p60, 2-3 frame delay with stabilization off). But needs Media Mod ($80 extra) and too expensive without sponsorship.
Insta360 X5 is not viable for rear-view
Webcam mode = 30fps with stitching latency. HDMI requires manual menu navigation (violates always-on reg). Pursue for content creation only.
